## 1. Introduction

### What is Renderspace Project Intelligence?

Renderspace Project Intelligence is an internal tool designed for digital agency project management. It provides:

- **Project Retrospectives**: Track and analyze completed projects
- **Financial Metrics**: Monitor budgets, margins, and profitability
- **Team Analytics**: Measure team performance and utilization
- **AI-Powered Tools**: Estimate projects and generate Jira tasks automatically

### Key Benefits

- Make data-driven decisions for future projects
- Learn from past project performance
- Improve estimation accuracy over time
- Streamline project planning with AI assistance

## 2. Getting Started

### Accessing the Application

1. Open your web browser (Chrome, Firefox, Safari, or Edge recommended)
2. Navigate to the application URL
3. The Dashboard will load as your home screen

### Navigation

The left sidebar contains all main sections:

| Icon | Section | Description |
|------|---------|-------------|
| Dashboard | Dashboard | Overview and key metrics |
| Folder | Projects | Project list and details |
| Chart | Analytics | Data visualizations |
| Users | Team Performance | Team member metrics |
| Chat | AI Assistant | AI-powered chat help |
| Calculator | Estimator | AI project estimation |
| List | PM Helper | Jira task generator |

### Edit Mode

At the bottom of the sidebar, you'll see either:
- **"View Only"** - You can browse but not edit data
- **"Edit Mode On"** - You can modify project data

To enable edit mode, click "View Only" and enter the password.

## 4. Projects

The Projects section is where you manage all project data and retrospectives.

### Projects List View

#### Viewing Projects

The main view displays all projects in a table format:

| Column | Description |
|--------|-------------|
| Project Name | Client and project title |
| Status | Draft, Active, or Completed |
| Health | On Track, At Risk, or Over Budget |
| Budget | Estimated vs actual budget |
| Margin | Profit margin percentage |
| Hours | Estimated vs actual hours |

#### Filtering and Sorting

- **Search**: Type in the search box to find projects by name
- **Status Filter**: Filter by Draft, Active, or Completed
- **Sort**: Click column headers to sort

#### Adding a New Project

1. Click the **"+ Add Project"** button
2. Fill in the project details form
3. Click **"Save"** to create the project

### Project Detail View

Click on any project to view its full details.

#### Overview Tab

Summary information including:
- Client name and project description
- Project type and CMS used
- Status and health indicators
- Key dates (start, end, deadline)

#### Hours Breakdown

Detailed hours tracking by profile:

| Profile | Description |
|---------|-------------|
| UX | User experience research and design |
| UI | Visual/interface design |
| DESIGN | General design work |
| DEV | Development and coding |
| PM | Project management |
| CONTENT | Content creation and entry |
| ANALYTICS | Analytics setup and tracking |

For each profile, you can see:
- **Estimated Hours**: Initial estimate
- **Actual Hours**: Hours actually spent
- **Variance**: Difference (positive = under, negative = over)

#### Financial Summary

- **Estimated Budget**: Initial project budget
- **Actual Cost**: Real costs incurred
- **Revenue**: Amount billed to client
- **Margin**: Profit margin (Estimated vs Actual)

#### Retrospective Notes

Free-text area for recording:
- What went well
- What could be improved
- Lessons learned
- Client feedback

#### Editing Project Data

1. Ensure Edit Mode is enabled (see sidebar)
2. Click on any editable field
3. Make your changes
4. Changes save automatically or click "Save"

## 8. Estimator

The AI-powered Estimator helps create accurate project estimates based on historical data.

### Creating an Estimate

#### Step 1: Enter Project Brief

Paste or type the project brief/requirements. Include:
- Project description
- Key features and functionality
- Number of pages/templates
- Any integrations needed
- Timeline constraints

**Or upload a document:**
- Click the upload area
- Select a PDF, Word, or text file
- The content will be extracted automatically

#### Step 2: Select Project Type

Choose from:
- Website
- Web Application
- E-commerce
- Mobile App
- Other

#### Step 3: Select CMS (if applicable)

Choose the content management system:
- WordPress
- Webflow
- Shopify
- Custom
- None
- Other options

#### Step 4: Add Integrations

List any third-party integrations:
- Payment gateways
- CRM systems
- Marketing tools
- APIs

#### Step 5: Generate Estimate

Click **"Generate Estimate"** and wait for the AI to analyze.

### Understanding the Estimate

#### Hours Breakdown

The estimate provides three-point estimation:

| Estimate | Description |
|----------|-------------|
| Optimistic | Best-case scenario (everything goes smoothly) |
| Realistic | Most likely scenario (normal conditions) |
| Pessimistic | Worst-case scenario (complications arise) |

Hours are broken down by profile:
- UX hours
- UI hours
- Development hours
- PM hours
- Content hours
- Analytics hours

#### Suggested Pricing

Based on realistic hours, see pricing at different hourly rates:
- Low rate (€60/hour)
- Medium rate (€70/hour)
- High rate (€80/hour)

#### Confidence Level

The AI provides a confidence rating:
- **High**: Similar projects in history, reliable estimate
- **Medium**: Some comparable data, moderate confidence
- **Low**: Limited data, treat as rough estimate

#### Risk Factors

Identified risks that could affect the estimate:
- Technical complexity
- Unclear requirements
- Tight timeline
- New technology
- Integration challenges

#### Recommendations

AI-generated suggestions:
- Suggested templates/features to include
- Potential scope clarifications needed
- Timeline recommendations

### Saving Estimates

1. Review the generated estimate
2. Click **"Save Estimate"** to store it
3. Access saved estimates in the History tab

### Estimate History

View all previous estimates:
- Search by project name
- See estimate accuracy after project completion
- Learn from past estimation patterns

### Providing Feedback

After a project completes, provide feedback:
- **Helpful**: Estimate was accurate
- **Not Helpful**: Estimate was significantly off

This improves future AI estimates.

## 9. PM Helper

The PM Helper generates Jira-ready task breakdowns from project offers.

### Creating Task Breakdowns

#### Step 1: Select Output Language

Choose the language for generated tasks:
- **English**: Tasks in English
- **Slovenian**: Tasks in Slovenian

#### Step 2: Upload or Paste Offer

**Option A - Upload File:**
1. Click the upload area or drag-drop a file
2. Supported formats: PDF, Word (.docx), Text (.txt)
3. Wait for parsing to complete

**Option B - Paste Text:**
1. Paste offer text directly into the text area
2. You can also type or edit manually

#### Step 3: Add Notes (Optional)

In the Additional Notes field, add:
- Focus areas
- Specific requirements
- Constraints or preferences

#### Step 4: Generate Tasks

1. Click **"Generate Tasks"**
2. Wait 30-60 seconds for AI processing
3. Tasks will appear in the right panel

### Understanding Generated Tasks

#### Task Types

| Type | Color | Description |
|------|-------|-------------|
| Epic | Purple | Large phases/features |
| Story | Green | User-facing functionality |
| Task | Blue | Technical work items |
| Subtask | Gray | Smaller pieces |

#### Task Properties

Each task includes:
- **Summary**: Task title
- **Description**: Details with acceptance criteria
- **Priority**: Highest, High, Medium, Low, Lowest
- **Labels**: Phase and role labels

#### Task Phases

Tasks are organized into phases:
1. Discovery
2. UX (wireframes per page)
3. UI Design (designs per page)
4. Development (per feature)
5. Content
6. QA/Testing
7. Launch

### Editing Tasks

Click any task to expand and edit:
- Modify summary and description
- Change task type or priority
- Edit labels
- Delete unwanted tasks

### Selecting Tasks for Export

- **Select All**: Check all tasks
- **Deselect All**: Uncheck all tasks
- **Individual**: Click checkbox per task

### Exporting to Jira

#### CSV Export

1. Select tasks to export (checkboxes)
2. Click **"Download CSV"** or **"Copy CSV"**
3. Import into Jira using CSV import

#### Jira Import Steps

1. In Jira: Project > Import Issues
2. Select CSV source
3. Upload the downloaded file
4. Map columns to Jira fields
5. Complete import

### Saving Work

- **Save**: Store generation in History
- **Template**: Save as reusable template

### History Tab

View and reload previous generations:
- Click **"Load"** to restore tasks
- Click trash icon to delete

### Templates Tab

Create and use task templates:
1. Generate tasks
2. Click **"Template"** button
3. Name your template
4. Reuse for similar projects

## 11. Troubleshooting

### Common Issues

#### Page Won't Load

1. Check your internet connection
2. Try refreshing the page (Ctrl/Cmd + R)
3. Clear browser cache
4. Try a different browser

#### Data Not Saving

1. Ensure Edit Mode is enabled
2. Check for error messages
3. Verify all required fields are filled
4. Try refreshing and re-entering data

#### AI Features Not Working

1. Check internet connection
2. Try with shorter input text
3. Wait a moment and try again
4. Check if the service is available

#### File Upload Failing

1. Ensure file is PDF, DOCX, or TXT
2. Check file size (keep under 10MB)
3. Try copying text and pasting instead
4. Ensure file isn't password-protected

#### CSV Import Issues in Jira

1. Verify column headers match Jira fields
2. Check that issue types exist in your project
3. Remove special characters that might break CSV
4. Try importing in smaller batches

### Getting Help

If you encounter issues not covered here:

1. Check browser developer console (F12) for errors
2. Try a different browser
3. Contact the development team
4. Report issues with steps to reproduce
